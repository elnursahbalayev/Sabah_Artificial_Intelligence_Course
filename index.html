What is Classification?

In machine learning, we distinguish between two main types of supervised learning tasks:

Regression (Week 4)

Predicts continuous values ‚Äî numbers that can take any value within a range.

Examples: house prices ($150,000), temperature (23.5¬∞C), stock prices

Classification (Today)

Predicts discrete categories ‚Äî labels from a fixed set of possible outcomes.

Examples: spam/not spam, disease/healthy, cat/dog/bird

The Sigmoid Function

The sigmoid function (also called the logistic function) is a mathematical function that maps any real number to a value between 0 and 1. This makes it perfect for representing probabilities.

œÉ(z) = 1 / (1 + e-z)
The Sigmoid Function ‚Äî where e ‚âà 2.718 (Euler's number)

Understanding the Formula

Let's break down what happens for different values of z:

When z ‚Üí -‚àû

e-z becomes very large, so 1/(1+large) ‚âà 0

When z = 0

œÉ(z) = 0.5

e0 = 1, so 1/(1+1) = 0.5

When z ‚Üí +‚àû

œÉ(z) ‚Üí 1

e-z becomes very small, so 1/(1+small) ‚âà 1

The Logistic Regression Model

Now we have all the pieces. Logistic regression combines the linear model from last week with the sigmoid function to create a classifier that outputs probabilities:

1

Step 1: Compute Linear Combination

First, we calculate z = w¬∑x + b, exactly like linear regression. The weight w and bias b are learned from training data.

2

Step 2: Apply Sigmoid Function

We pass z through the sigmoid to get a probability: P(y=1|x) = œÉ(z). This ensures output is always between 0 and 1.

3

Step 3: Interpret as Probability

The output represents P(y=1|x), the probability that the input x belongs to class 1. Note: P(y=0|x) = 1 - P(y=1|x).

P(y=1|x) = œÉ(w¬∑x + b) = 1 / (1 + e-(w¬∑x + b))
Complete Logistic Regression Model

üîë Role of Weight (w)

Controls the steepness of the sigmoid curve. Larger |w| means sharper transition between classes. The sign determines direction (positive w: higher x ‚Üí higher probability).

üîë Role of Bias (b)

Controls where the decision boundary is located. Changing b shifts the sigmoid curve left or right along the x-axis without changing its shape.

‚è±Ô∏è ~1.5 min | Emphasize the pipeline: "We first compute the same linear combination as before, then squeeze it through sigmoid to get a probability." Make sure students understand the roles of w and b.

Decision Boundary

The sigmoid gives us a probability, but in practice we often need to make a definite decision: Class 0 or Class 1. We do this by choosing a threshold (typically 0.5):

≈∑ = 1 if P(y=1|x) ‚â• 0.5, otherwise ≈∑ = 0
Decision Rule ‚Äî classify based on threshold

The decision boundary is the point (or line/surface in higher dimensions) where the model is exactly 50% confident. This occurs when œÉ(z) = 0.5, which happens when z = 0:

w¬∑x + b = 0 ‚Üí x = -b/w

üí° Adjusting the Threshold

The default 0.5 threshold isn't always optimal. In medical diagnosis, we might lower it to 0.3 to catch more true positives (higher recall). In spam detection, we might raise it to 0.7 to avoid false positives. The right threshold depends on the cost of different types of errors.

Binary Cross-Entropy Loss

To train our model (find optimal w and b), we need a loss function that measures how wrong our predictions are. For classification, we use Binary Cross-Entropy (also called Log Loss):

L = -[y¬∑log(≈∑) + (1-y)¬∑log(1-≈∑)]
Binary Cross-Entropy Loss ‚Äî where y is actual (0 or 1), ≈∑ is predicted probability

Why This Formula?

The formula looks complex, but it's actually elegant. Because y is either 0 or 1, only one term is active at a time:

When actual y = 1 (positive case)

The formula simplifies to: Loss = -log(≈∑)

  • If ≈∑ ‚Üí 1 (correct!), loss ‚Üí 0
  • If ≈∑ ‚Üí 0 (wrong!), loss ‚Üí ‚àû

When actual y = 0 (negative case)

The formula simplifies to: Loss = -log(1-≈∑)

  • If ≈∑ ‚Üí 0 (correct!), loss ‚Üí 0
  • If ≈∑ ‚Üí 1 (wrong!), loss ‚Üí ‚àû

üîß Explore the Loss Function

Predicted probability (≈∑) 0.50
Loss = 0.693
-log(0.50) = 0.693

üéØ Key Property: Punishes Confident Mistakes

Notice how the loss explodes when the model is confidently wrong (predicting 0.99 when actual is 0). This is intentional ‚Äî BCE heavily penalizes overconfident incorrect predictions, which helps the model learn to be well-calibrated.

Python Implementation: Core Functions

Let's implement logistic regression step by step. First, the fundamental building blocks:

import numpy as np # 1. The Sigmoid Function def sigmoid(z): """Transform any value to range [0,1]""" return 1 / (1 + np.exp(-z)) # Test it print(sigmoid(0)) # 0.5 print(sigmoid(2)) # 0.88 print(sigmoid(-2)) # 0.12
Line 1

NumPy provides np.exp() for efficient exponential calculations, even on arrays.

Line 5

Direct implementation of œÉ(z) = 1/(1+e‚Åª·∂ª). Works with single numbers or entire arrays!

Lines 8-10

Testing: z=0‚Üí0.5 (boundary), z=2‚Üí0.88 (likely class 1), z=-2‚Üí0.12 (likely class 0)

# 2. Predict Probability def predict_proba(X, w, b): """Get P(y=1|x) for each sample""" z = np.dot(X, w) + b # Linear return sigmoid(z) # Squash # 3. Predict Class Labels def predict(X, w, b, threshold=0.5): """Convert probabilities to 0/1""" probs = predict_proba(X, w, b) return (probs >= threshold).astype(int)
Line 4

np.dot(X, w) computes the linear combination for all samples at once. This is vectorized and very fast.

Line 5

Pass through sigmoid to get probabilities. The two-step process: linear ‚Üí non-linear.

Line 11

Compare to threshold and convert boolean to int. Default 0.5 means "more likely than not."

‚è±Ô∏è ~2 min | Walk through the code line by line. Emphasize vectorization ‚Äî this handles thousands of samples efficiently. Point out the two-step process in predict_proba.

Python Implementation: Loss & Training

Now let's implement the loss function and see a complete example:

# 4. Binary Cross-Entropy Loss def bce_loss(y_true, y_pred): """Compute average loss over samples""" # Prevent log(0) with small epsilon eps = 1e-15 y_pred = np.clip(y_pred, eps, 1-eps) # BCE formula loss = -(y_true * np.log(y_pred) + (1-y_true) * np.log(1-y_pred)) return np.mean(loss)
Lines 5-6

np.clip ensures y_pred stays in (Œµ, 1-Œµ) to avoid log(0) = -‚àû which would crash our code.

Lines 9-10

Direct implementation of L = -[y¬∑log(≈∑) + (1-y)¬∑log(1-≈∑)]. NumPy handles element-wise operations.

Line 11

np.mean averages over all samples to get a single loss value for the batch.

Complete Example: Student Pass/Fail

# Sample data: hours studied X = np.array([[1], [2], [3], [5], [6], [7]]) # Features y = np.array([0, 0, 0, 1, 1, 1]) # Labels (fail/pass) # Pretrained weights (in practice, learned via gradient descent) w = np.array([1.5]) # Positive: more hours ‚Üí higher prob b = -6.0 # Shifts boundary to ~4 hours # Make predictions probs = predict_proba(X, w, b) predictions = predict(X, w, b) print("Hours: ", X.flatten()) # [1, 2, 3, 5, 6, 7] print("Probability: ", probs.round(2)) # [0.01, 0.05, 0.18, 0.82, 0.95, 0.99] print("Predicted: ", predictions) # [0, 0, 0, 1, 1, 1] print("Actual: ", y) # [0, 0, 0, 1, 1, 1] print("Loss: ", bce_loss(y, probs).round(3)) # ~0.124

Key Takeaways

Sigmoid Function

œÉ(z) = 1/(1+e‚Åª·∂ª) transforms any real number into range [0,1], making it perfect for representing probabilities. œÉ(0) = 0.5 is the natural decision point.

Logistic Regression

P(y=1|x) = œÉ(w¬∑x + b) combines linear model with sigmoid. Weight w controls steepness, bias b shifts the decision boundary.

Decision Boundary

The point where P = 0.5, defined by w¬∑x + b = 0. We classify based on which side of this boundary samples fall.

Binary Cross-Entropy

L = -[y¬∑log(≈∑) + (1-y)¬∑log(1-≈∑)] penalizes confident wrong predictions heavily, creating a convex optimization surface.

üîó Connection to Deep Learning

The sigmoid function you learned today is used as an activation function in neural networks. Understanding it deeply prepares you for more complex architectures!
